# 🛡️ Botium Toys Security Audit

## 📌 Overview
This project presents an internal security audit for Botium Toys, a fictional company used in the Google Cybersecurity Professional Certificate.

The audit evaluates the organization's current security posture, identifies risks, assesses controls, and provides recommendations to improve compliance and reduce vulnerabilities.

---

## 🎯 Objectives
- Assess current security controls
- Identify risks and vulnerabilities
- Evaluate compliance gaps (PCI DSS, GDPR, SOC)
- Recommend improvements to strengthen security posture

---

## 🔍 Key Findings
- High risk score: **8/10**
- Lack of critical controls (least privilege, encryption, IDS)
- Weak compliance with industry standards
- No proper access control or data protection mechanisms

---

## 🛠️ Controls Assessment Summary

| Control | Status |
|--------|--------|
| Firewall | ✅ Implemented |
| Antivirus | ✅ Implemented |
| Encryption | ❌ Not Implemented |
| Least Privilege | ❌ Not Implemented |
| IDS | ❌ Not Implemented |
| Backup Systems | ❌ Not Implemented |

---

## 📋 Compliance Gaps

- **PCI DSS:** Not compliant (payment data not secured)
- **GDPR:** Not compliant (EU data protection missing)
- **SOC:** Weak internal controls

---

## ⚠️ Risk Summary
The organization faces:
- Data breaches
- Regulatory fines
- Business disruption
- Loss of customer trust

---

## ✅ Recommendations

### High Priority
- Implement least privilege access control
- Enforce strong password policies
- Introduce encryption for sensitive data
- Deploy IDS/monitoring tools

### Medium Priority
- Develop disaster recovery plan
- Implement backup solutions
- Introduce security awareness training
